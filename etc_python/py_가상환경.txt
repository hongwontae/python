1. which python
=> python이 어떤 환경에서 돌아가고 있는지 알 수 있는 명령어입니다.
=> C:\Users\dnjsx\Desktop\Python\.venv\Scripts\python.exe .venu 내의 경로라면 가상환경이 python이 기본으로
설정된 상태입니다.

* python 실행 파일이 가상 환경 내부에 설치된 경우
=> 가상 환경을 설정하면 해당 디렉터리 내부에 독립적인 python 실행 파일이 복사되거나 심볼릭 링크로 연결된다는
뜻입니다.

* 가상 환경 구조
.venv/
├── Include/
├── Lib/
│   ├── site-packages/
├── Scripts/ (Windows) 또는 bin/ (macOS/Linux)
│   ├── python.exe  <-- 가상 환경 내부의 Python 실행 파일
│   ├── activate    <-- 가상 환경 활성화 스크립트
1. Scripts/python.exe (Windows)
=> 가상 환경 내부에 있는 python 실행 파일입니다.


2. 가상 환경
=> 가상 환경에서는 python 시스템은 시스템 python과 격리된 상태에서 동작합니다.
=> 패키지를 설치하면 시스템 python의 site-packages가 아닌 가상 환경 Lib/site-packages에 설치됩니다.


* 동작 방식
=> Windows에서는 현재 디렉터리에서 실행 파일을 먼저 찾습니다.
=> 만약 py_step 폴더에 Python 실행 파일이 있거나, .venv 폴더가 동일한 레벨에 있고 py_step에서 Python을 
실행할 경우, 가상 환경 .venv/Scripts/python.exe이 우선적으로 사용될 가능성이 큽니다.
